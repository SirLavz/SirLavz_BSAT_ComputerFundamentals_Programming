A comprehensive guide to naming conventions in Java

What is a Java Identifier?

In Java, an identifier is a unique name assigned to various programming elements. These include variables, classes, methods, packages, and interfaces. They act as a way to identify and differentiate between these components within a program.

Rules for Naming Java Identifiers

To be considered a valid identifier, a name must follow a specific set of rules:

  • Allowed characters include alphanumeric characters (A-Z, a-z, 0-9), the dollar sign ($), and the underscore (_).
  • An identifier cannot start with a digit (0-9).
  • Java is a case-sensitive language, so myVariable and MyVariable are treated as different identifiers.
  • Reserved words (keywords and literals) cannot be used as identifiers.
  • There is no official length limit for an identifier, but it's generally recommended to keep them at an optimum length (e.g., 4-15 characters) for readability.

Example of Identifiers in Code

In the following code snippet, there are five identifiers:

class Test {
    public static void main(String args[]) {
        int a = 10;
    }
}
  • Test: A class name
  • main: A method name
  • String: A predefined class name
  • args: A variable name
  • a: A variable name

Valid and Invalid Identifier Examples

Valid Identifiers

  • MyVariable
  • _myVariable
  • $amount
  • sum_of_array
  • geeks123

Invalid Identifiers

  • My Variable (contains a space)
  • 123geeks (starts with a digit)
  • a+c (contains an invalid character)
  • while (is a reserved keyword)

Java Reserved Words

Java has 53 reserved words that cannot be used as identifiers. These include:

  • Keywords (50): Words that define functionalities (e.g., class, public, static, void).
  • Literals (3): The values true, false, and null.
