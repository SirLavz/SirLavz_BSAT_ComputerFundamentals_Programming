Module 1: Complete Discussion of Topics

Topic 1.1: Introduction to Computers

A computer is an electronic device that manipulates information, or data. It has the ability to store, retrieve, and process data. Computers have evolved dramatically over time, moving through distinct generations, each marked by significant technological advancements.

  • First Generation (1940s-1950s): Characterized by vacuum tubes for circuitry and magnetic drums for memory. They were enormous, consumed vast amounts of electricity, and generated a lot of heat. Programming was done in machine language.
  • Second Generation (1950s-1960s): Introduced transistors, which replaced vacuum tubes, making computers smaller, faster, cheaper, and more energy-efficient. High-level programming languages like COBOL and FORTRAN emerged.
  • Third Generation (1960s-1970s): The invention of the integrated circuit (IC) was the hallmark. This allowed many transistors to be placed on a single silicon chip, further shrinking computers and increasing their speed and efficiency. Operating systems were introduced.
  • Fourth Generation (1970s-present): Marked by the microprocessor, which brought the entire CPU onto a single chip. This led to the development of personal computers (PCs). The internet and graphical user interfaces (GUIs) became widespread.
  • Fifth Generation (Present and Beyond): Focuses on Artificial Intelligence (AI), parallel processing, and natural language understanding. Quantum computing is an emerging area in this generation.

Computers come in various forms, each suited for different tasks:

  • Supercomputers: The fastest and most powerful computers, used for complex calculations and simulations (e.g., weather forecasting, aerospace design).
  • Mainframe Computers: Large, powerful computers used by large organizations for critical applications, data processing, and transaction handling (e.g., airline reservation systems).
  • Minicomputers (Mid-range Servers): Smaller than mainframes but more powerful than microcomputers, often used for specific tasks in businesses.
  • Microcomputers (Personal Computers): The most common type, including desktops, laptops, tablets, and smartphones.

The fundamental distinction in a computer system is between hardware (the physical components) and software (the instructions that tell the hardware what to do).

Topic 1.2: Computer Hardware

Hardware refers to the tangible, physical parts of a computer system.

  • Input Devices: Allow users to feed data into the computer. Examples include keyboards (for text input), mice (for navigation and selection), scanners (for converting physical documents into digital images), and microphones (for audio input).
  • Output Devices: Display or present processed data from the computer. Common examples are monitors (visual display), printers (hard copies), and speakers (audio output).
  • Central Processing Unit (CPU): Often called the "brain" of the computer, the CPU performs all calculations and executes program instructions. It consists of the Arithmetic Logic Unit (ALU) for arithmetic and logical operations, and the Control Unit (CU) for managing and coordinating operations.
  • Memory:
    • Random Access Memory (RAM): Volatile memory used for temporary storage of data and programs currently being used. Data is lost when the power is off.
    • Read-Only Memory (ROM): Non-volatile memory that stores essential boot-up instructions. Data persists even when power is off.
    • Cache Memory: A small, very fast memory that stores frequently accessed data for quicker retrieval by the CPU.
  • Storage Devices: Used for long-term storage of data and programs.
    • Hard Disk Drives (HDD): Traditional magnetic storage, slower but generally cheaper and with higher capacity.
    • Solid State Drives (SSD): Newer, faster storage that uses flash memory, more durable but typically more expensive.
    • USB Drives (Flash Drives): Portable, small storage devices.
    • Cloud Storage: Data stored on remote servers and accessed over the internet (e.g., Google Drive, Dropbox).

Topic 1.3: Computer Software

Software is a set of instructions, data, or programs used to operate computers and execute specific tasks. It's the intangible part of the computer.

  • System Software: Designed to operate the computer hardware and to provide a platform for running application software.
    • Operating Systems (OS): The most crucial system software, managing computer hardware and software resources (e.g., Windows, macOS, Linux, Android, iOS).
    • Utility Programs: Assist in maintaining and optimizing computer performance (e.g., antivirus software, disk cleanup tools).
    • Device Drivers: Software that enables communication between the operating system and hardware devices.
  • Application Software: Designed to perform specific tasks for the user. Examples include word processors (Microsoft Word), spreadsheets (Excel), database management systems (SQL Server), and specialized software like aviation-specific software (e.g., flight simulators, air traffic control software, aircraft maintenance systems).

Topic 1.4: Operating Systems

An Operating System (OS) is the software that manages computer hardware and software resources and provides common services for computer programs. It acts as an intermediary between the user and the computer hardware.

Key functions of an OS include:

  • Memory Management: Allocating and deallocating memory space for programs.
  • Process Management: Managing the execution of multiple programs (processes).
  • File Management: Organizing, storing, retrieving, and deleting files and folders.
  • Device Management: Managing input/output devices.
  • Security: Protecting system resources and user data.

Common types of OS:

  • Microsoft Windows: The most widely used desktop OS.
  • macOS: Apple's operating system for its Macintosh computers.
  • Linux: An open-source, highly customizable OS, popular in servers and embedded systems.
  • Mobile OS: Operating systems for smartphones and tablets, such as Android and iOS.

Students will learn basic OS commands for file and folder creation, deletion, copying, and moving, which are fundamental for managing programming projects.

Topic 1.5: Computer Networks and Internet

A computer network is a group of interconnected computers and devices that can share resources and data. Understanding networks is vital in air transportation, where real-time data exchange is critical.

  • Network Topologies: The physical or logical arrangement of connected devices.
    • Local Area Network (LAN): Connects devices within a limited area, like a building or campus (e.g., PhilSCA computer lab network).
    • Wide Area Network (WAN): Connects devices over a large geographical area, often using telecommunication lines (e.g., connecting PhilSCA to other institutions or global airline networks).
  • Network Devices: Hardware components that facilitate network communication.
    • Routers: Direct data packets between different networks.
    • Switches: Connect devices within a single network segment.
    • Hubs: Basic devices that connect multiple Ethernet devices together.
  • Protocols: Sets of rules that govern how data is transmitted over a network.
    • TCP/IP (Transmission Control Protocol/Internet Protocol): The foundational protocol suite for the internet.
    • HTTP (Hypertext Transfer Protocol): Used for transferring web pages.
    • FTP (File Transfer Protocol): Used for transferring files between computers.
  • Introduction to the Internet: A global network of interconnected computer networks that uses TCP/IP to serve billions of users worldwide. It enables services like email, web browsing, and online communication.
Relevance in Air Traffic Management (ATM) and airline operations:
  • Air Traffic Control (ATC): Relies heavily on robust networks for radar data, flight plan information, and communication between controllers and pilots.
  • Airline Reservation Systems: Global networks allow passengers to book flights, manage reservations, and check-in from anywhere.
  • Flight Planning and Operations: Pilots and dispatchers use networked systems to access weather data, NOTAMs (Notices to Airmen), flight plans, and operational updates.
  • Aircraft Communications: ACARS (Aircraft Communications Addressing and Reporting System) uses data links for communication between aircraft and ground stations.
  • Maintenance and Logistics: Networked systems track aircraft parts, maintenance schedules, and supply chains.

Module 1: Laboratory Activity

Laboratory Activity 1: Exploring Your Operating System and Network Basics

Objective: To familiarize students with their computer's operating system environment, basic file management, and fundamental network configurations.

Materials:

  • A computer with a working operating system (Windows, macOS, or Linux).
  • Internet access.
Instructions:

1. System Information (15 minutes):

  • Find and record your computer's operating system version (e.g., Windows 10, macOS Ventura, Ubuntu 22.04).
  • Identify your CPU type (e.g., Intel Core i7, AMD Ryzen 5) and installed RAM (e.g., 8GB, 16GB).
  • Determine the total storage capacity of your primary drive and the amount of free space.
  • How to find:
    • Windows: Right-click "This PC" or "My Computer" > "Properties" or go to Settings > System > About.
    • macOS: Click the Apple menu > "About This Mac."
    • Linux: Open a terminal and use commands like `lsb_release -a`, `lscpu`, `free -h`, `df -h`.

2. File Management Practice (20 minutes):

  • Create a new folder on your Desktop named `CFPJ101_Lab1_[YourLastName]`.
  • Inside this folder, create three subfolders: `Documents`, `Images`, `Temporary`.
  • Create a simple text file (e.g., using Notepad on Windows, TextEdit on macOS, Gedit on Linux) named `my_info.txt` inside the `Documents` subfolder. Type your full name, student ID, and course code (CFPJ 101) into this file and save it.
  • Copy `my_info.txt` from `Documents` to `Temporary`.
  • Rename the `Temporary` folder to `Backup`.
  • Delete the `Images` folder.
  • Take a screenshot of your `CFPJ101_Lab1_[YourLastName]` folder showing its contents after these operations.

3. Network Configuration Check (25 minutes):

  • Open your computer's network settings.
  • Identify if you are connected to a Wi-Fi network or a wired (Ethernet) connection.
  • Find and record your computer's IP Address (IPv4).
  • Find and record your Default Gateway address.
  • Find and record your DNS Server address(es).
  • Ping Test:
    • Open a Command Prompt (Windows), Terminal (macOS/Linux).
    • Type `ping google.com` and press Enter.
    • Record the average response time (latency).
    • Explain what a "ping" command does and what the response time indicates.
  • How to find network info:
    • Windows: `ipconfig` in Command Prompt or Settings > Network & Internet > Status > View your network properties.
    • macOS: System Settings > Network > Select your active connection > Details/Advanced.
    • Linux: `ip a` or `ifconfig` in Terminal.

4. Internet Research (20 minutes):

  • Using a web browser, research and find one specific example of how computer networks are critical for Air Traffic Control (ATC) operations.
  • Write a short paragraph (3-5 sentences) summarizing your findings, including the source (website URL). Save this paragraph in a new text file named `ATC_Network_Research.txt` inside your `CFPJ101_Lab1_[YourLastName]` folder.
Deliverables:
  1. A text file (e.g., `System_Info.txt`) containing the recorded system information (OS, CPU, RAM, Storage).
  2. The screenshot of your `CFPJ101_Lab1_[YourLastName]` folder.
  3. A text file (e.g., `Network_Info.txt`) containing your recorded network configuration (IP, Gateway, DNS) and the ping test results with explanation.
  4. The `ATC_Network_Research.txt` file.

Submission: Submit all four (4) files in a single compressed `.zip` or `.rar` archive named `CFPJ101_Lab1_[YourLastName].zip` through the designated online learning platform.
