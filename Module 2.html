Module 2: Fundamentals of Programming and Algorithmic Thinking

Week 1

Session 1: The Building Blocks of Code

This session introduces the fundamental elements of programming that form the basis of all code.

Topics Covered:

  • Introduction to Programming: What is a program? A brief overview of the difference between low-level and high-level languages.
  • Variables and Data Types: We'll define variables as named memory locations for storing data. We'll explore fundamental data types like Integer, Float, String, and Boolean.
  • Basic Operations: We'll discuss how to perform basic mathematical calculations using arithmetic operators (+, -, *, /) and how to assign values using the assignment operator (=). We'll also cover comparison operators (==, !=, <, >) for making decisions.

Homework/Activity (Before Next Session):

  • Write a short pseudocode program that takes two numbers as input, calculates their product and sum, and then prints both results.
  • Create a flowchart for a simple process like deciding whether to wear a jacket based on the temperature.

Session 2: Controlling the Flow of a Program

This session focuses on how to control the order and repetition of instructions in your code.

Topics Covered:

  • Conditional Statements (if, else if, else): We'll explain how these statements allow a program to make decisions and execute different code paths based on conditions. We'll also cover logical operators (AND, OR, NOT) for combining conditions.
  • Loops (for, while): We'll introduce the power of repetition. for loops are for a known number of repetitions, while while loops continue as long as a condition is true.
  • Functions: We'll define functions as reusable blocks of code. We'll discuss the benefits of using functions and the concepts of parameters (input) and return values (output).

Homework/Activity (Before Next Session):

  • Write a pseudocode program that uses a loop to print all odd numbers between 1 and 100.
  • Create a function in pseudocode called is_prime that takes a number and returns True if it's a prime number, and False otherwise.

Week 2

Session 3: The Art of Algorithmic Thinking

This session shifts focus from programming syntax to the crucial skill of problem-solving.

Topics Covered:

  • What is an Algorithm? We'll define an algorithm as a precise, step-by-step procedure for solving a problem, independent of any specific programming language.
  • The Problem-Solving Process: We'll formalize a four-step process for creating algorithms: understand the problem, devise a plan, execute the plan, and review/refine.
  • Introduction to Data Structures (Arrays/Lists): We'll introduce the concept of data structures as a way to organize data, with a focus on arrays or lists.

Homework/Activity (Before Next Session):

  • Given a list of names, write an algorithm (in pseudocode) to find the longest name in the list.
  • Research the difference between a "linear search" and a "binary search."

Session 4: Analyzing and Optimizing Algorithms

The final session covers how to measure and improve the efficiency of your algorithms.

Topics Covered:

  • Searching Algorithms: We'll dive into two common searching methods: Linear Search and Binary Search.
  • Sorting Algorithms (e.g., Bubble Sort): We'll introduce the concept of sorting and use a simple algorithm like bubble sort to illustrate the process.
  • Algorithm Analysis and Big O Notation ($O$): We'll explain Big O notation as a way to describe how an algorithm's performance scales with the size of its input, covering notations like $O(n)$ and $O(\log n)$.
